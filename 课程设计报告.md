## 三 系统方案和硬件设计

### 3.1 系统方案

系统采用"上位机浏览器 + 嵌入式Web服务器 + Linux GPIO控制"的三层架构：

```
┌─────────────┐     HTTP请求      ┌─────────────────┐     sysfs写入      ┌──────────────┐
│   PC浏览器   │ ◄──────────────► │  Python Web服务  │ ◄──────────────► │  Linux GPIO   │
│ (Chrome等)   │   TCP:8080       │  (开发板:8080)   │   /sys/class/    │  (LED/蜂鸣器)  │
└─────────────┘                   └─────────────────┘   leds, gpio      └──────────────┘
```

**工作流程**：
1. 用户在浏览器中访问开发板IP地址（http://192.168.1.100:8080）
2. 浏览器发送HTTP GET请求到Python Web服务器
3. 服务器解析请求URL，判断控制类型（LED/蜂鸣器/按键）
4. 服务器通过Linux sysfs接口读写GPIO，控制硬件
5. 服务器返回HTML页面，浏览器渲染显示

### 3.2 硬件设计

#### 3.2.1 开发板硬件平台
采用米尔科技Z-Turn开发板，核心芯片为Xilinx Zynq-7020，集成双核ARM Cortex-A9处理器和FPGA可编程逻辑。

**主要硬件参数**：
- 处理器：Zynq-7020 (XC7Z020)，双核ARM Cortex-A9 @ 666MHz
- 内存：1GB DDR3 (512MB x 2)
- 存储：4GB SD卡 + 128Mbit QSPI Flash
- 网络：10/100/1000M以太网 (HanRun HY911130A)
- 接口：USB OTG、USB UART、HDMI、CAN、JTAG
- 外设：RGB三色LED、2个用户LED、蜂鸣器、用户按键、4个拨码开关、温度传感器、G-Sensor(ADXL345)

#### 3.2.2 GPIO引脚分配

| 外设 | GPIO号 | 方向 | 控制方式 |
|------|--------|------|---------|
| usr_led1 | GPIO 0 | 输出 | sysfs brightness |
| usr_led2 | GPIO 9 | 输出 | sysfs brightness |
| led_r (实际绿色) | GPIO 114 | 输出 | sysfs brightness |
| led_g (实际蓝色) | GPIO 115 | 输出 | sysfs brightness |
| led_b (实际红色) | GPIO 116 | 输出 | sysfs brightness |
| 蜂鸣器 | GPIO 117 | 输出 | sysfs value (方波) |
| 按键K1 | GPIO 50 | 输入 | input event |

**注**：设备树中的LED颜色标签与实际硬件颜色不一致。led_r实际连接绿色LED，led_g实际连接蓝色LED，led_b实际连接红色LED。

#### 3.2.3 蜂鸣器控制原理

蜂鸣器通过GPIO 117控制。系统启动时，内核gpio-beeper驱动会占用该GPIO。为实现频率控制（不同音调），需要先解绑该驱动：

```
echo gpio-beep.3 > /sys/bus/platform/drivers/gpio-beeper/unbind
```

解绑后，通过sysfs GPIO接口直接控制：
- 导出GPIO：`echo 117 > /sys/class/gpio/export`
- 设为输出：`echo out > /sys/class/gpio/gpio117/direction`
- 高低电平切换产生方波：`echo 1 > /sys/class/gpio/gpio117/value` / `echo 0`

方波频率计算公式：`f = 1 / (2 × half_period)`

### 4.4 关键程序代码和解释

#### 4.4.1 LED控制函数

```python
def set_led(name, value):
    """控制LED亮灭
    name: LED名称(usr_led1/led_r等)
    value: 1=亮 0=灭
    """
    path = LED_MAP.get(name)
    if path:
        with open(path, 'w') as f:
            f.write('255' if value else '0')  # 255=最大亮度, 0=熄灭
        return True
    return False
```

**说明**：通过Linux LED子系统的sysfs接口控制。写入`/sys/class/leds/<name>/brightness`文件，值为0~255。

#### 4.4.2 蜂鸣器控制函数

```python
def set_buzzer(freq):
    """控制蜂鸣器频率
    freq: 频率(Hz)，0=停止
    """
    if freq == 0:
        os.system('/root/project/tone3 0')     # 停止
    else:
        os.system('/root/project/tone3 ' + str(freq) + ' 500')  # 频率+时长(ms)
    return True
```

**说明**：调用编译好的C程序`tone3`，通过直接操作GPIO 117产生方波。Python的sleep精度不够，因此用C实现精确的微秒级延时。

#### 4.4.3 蜂鸣器C程序核心代码

```c
// GPIO sysfs控制产生方波
for (i = 0; i < total; i++) {
    // 写入HIGH
    lseek(fd_val, 0, SEEK_SET);
    write(fd_val, "1", 1);
    usleep(half_us);    // 高电平持续时间

    // 写入LOW
    lseek(fd_val, 0, SEEK_SET);
    write(fd_val, "0", 1);
    usleep(half_us);    // 低电平持续时间
}
```

**说明**：通过sysfs接口 `/sys/class/gpio/gpio117/value` 快速切换GPIO电平，产生方波。不同频率对应不同的`half_us`值。

#### 4.4.4 按键检测函数

```python
def get_button():
    """读取按键状态
    返回: 1=按下, 0=松开, -1=无事件
    """
    f = os.open(BUTTON_DEV, os.O_RDONLY | os.O_NONBLOCK)
    data = os.read(f, 24)
    os.close(f)
    if len(data) >= 16:
        _, _, ev_type, ev_code, ev_value = struct.unpack('llhhi', data[:16])
        if ev_type == 1:  # EV_KEY事件
            return ev_value
    return -1
```

**说明**：通过Linux input子系统读取按键事件。`/dev/input/event2`是非阻塞读取，有按键事件时返回事件数据，无事件时立即返回。

## 五 系统制作和测试

### 5.1 制作过程

1. **硬件连接**：通过网线将开发板连接至交换机/路由器，PC通过网络访问开发板
2. **环境搭建**：通过串口登录开发板，配置网络（静态IP 192.168.1.100）
3. **驱动解绑**：解绑gpio-beeper驱动，释放GPIO 117用于蜂鸣器频率控制
4. **程序开发**：编写Python Web服务器和C蜂鸣器控制程序
5. **交叉编译**：在开发板上使用gcc编译tone3程序
6. **联调测试**：通过浏览器访问Web界面，逐项测试各功能

**遇到的困难及解决方案**：
1. **LED颜色不匹配**：设备树中led_r/g/b标签与实际硬件颜色不一致。通过实际点亮测试确认对应关系。
2. **蜂鸣器无法变频**：gpio-beeper驱动只支持开/关。解决方案：解绑驱动后直接操作GPIO。
3. **RGB LED亮度不足**：写入brightness值为1时太暗。改为写入255（最大亮度）。
4. **CSS与Python格式化冲突**：HTML中CSS的`100%`被Python的`%`格式化解释。改用字符串拼接代替模板格式化。
